## 1. 文档范围

本文档面向当前已落地的 WinForms 页面，覆盖：

- 登录与主界面；
- 用户与权限；
- Motion 监视与调试页面；
- PLC 状态、点位监视、调试与配置页面；
- 当前报警、报警历史、运行日志页面。

## 2. 角色说明

| 角色 | 说明 |
|------|------|
| `Operator` | 以监视为主，不开放高风险控制与配置 |
| `Engineer` | 可进行配置、控制和调试 |
| `Am` | 具备全部权限，含用户与权限管理 |

## 3. 登录与主界面

### 3.1 登录

- 默认管理员账号：`am`
- 默认密码：`am123`

操作步骤：

1. 启动程序；
2. 在登录页输入账号密码；
3. 登录成功后进入主界面；
4. 取消登录则直接退出程序。

### 3.2 主界面导航

主界面结构：

- 左侧一级导航；
- 中间二级导航；
- 右侧页面工作区；
- 底部状态栏；
- 右下报警按钮可打开活动报警抽屉。

### 3.3 用户菜单

右上角头像菜单支持：

- 切换用户；
- 修改密码（当前仍为占位）；
- 退出登录。

## 4. 系统管理页面

## 4.1 用户管理 `System.User`

适用角色：`Am`

主要功能：

- 查询用户；
- 新增用户；
- 编辑用户；
- 删除用户；
- 启用 / 禁用用户；
- 重置密码。

推荐操作流程：

1. 输入关键字筛选用户；
2. 点击“新增用户”或在列表中编辑；
3. 保存后观察顶部消息与列表刷新；
4. 如需重置密码，使用“重置密码”操作。

## 4.2 权限分配 `System.Permission`

适用角色：`Am`

主要功能：

- 选择目标用户；
- 按模块切换页面权限；
- 全选 / 清空当前模块权限；
- 恢复默认权限；
- 保存权限。

推荐操作流程：

1. 点击“选择用户”；
2. 选择模块；
3. 勾选页面；
4. 点击“保存”；
5. 如保存的是当前登录用户，导航会自动刷新。

## 4.3 登录日志 `System.LoginLog`

适用角色：`Am`

主要功能：

- 查询登录历史；
- 按全部 / 成功 / 失败 / 今日筛选；
- 按时间范围和关键字查询；
- 分页浏览。

## 5. Motion 页面

## 5.1 DI 监视 `Motion.DI`

适用角色：`Operator / Engineer / Am`

主要功能：

- 按控制卡筛选；
- 关键字搜索；
- 分页查看 DI 点位；
- 选中后在右侧查看详情。

使用说明：

1. 点击“选择控制卡”；
2. 输入点位关键字；
3. 左侧选择 DI 卡片；
4. 在右侧查看点位名称、逻辑位、状态等详情。

## 5.2 DO 监视 `Motion.DO`

适用角色：`Operator / Engineer / Am`

与 DI 页面用法基本一致，主要用于查看输出点位状态。

## 5.3 多轴总览 `Motion.Monitor`

适用角色：`Operator / Engineer / Am`

主要功能：

- 选择控制卡；
- 搜索轴；
- 查看轴总数、报警数、运动中数量、就绪数量；
- 左侧查看轴卡片，右侧查看选中轴详情。

## 5.4 轴控制 `Motion.Axis`

适用角色：`Engineer / Am`

主要功能：

- 选择目标轴；
- 执行使能、失能、停止、回零等动作；
- 输入速度、绝对位置、相对距离后执行动作；
- 右侧持续查看该轴实时状态。

操作建议：

1. 先确认当前选中轴正确；
2. 执行使能与回零前先检查报警与限位；
3. 绝对定位 / 相对定位前检查参数值是否正确；
4. 如动作失败，查看状态栏与运行日志。

## 5.5 执行器控制 `Motion.Actuator`

适用角色：`Engineer / Am`

主要功能：

- 按类型过滤执行器；
- 搜索执行器；
- 左侧选中执行器；
- 中间执行动作；
- 右侧查看反馈与详情。

## 6. PLC 页面

## 6.1 通讯状态 `PLC.Status`

适用角色：

- 监视：`Operator / Engineer / Am`
- 扫描控制：`Engineer / Am`

主要功能：

- 查看全部 PLC 站在线状态；
- 搜索 PLC 站；
- 手动刷新；
- 单轮扫描；
- 启动 / 停止扫描；
- 左侧选择站，右侧查看详情。

使用建议：

1. 若 PLC 页面无数据，先进入本页查看站是否在线；
2. 若站离线，先检查协议、IP、端口、站号；
3. 若需要联调，工程师可手动单轮扫描验证。

## 6.2 点位监视 `PLC.Monitor`

适用角色：`Operator / Engineer / Am`

主要功能：

- 按 PLC 筛选；
- 按分组筛选；
- 关键字搜索点位；
- 查看点位值、质量、更新时间；
- 选中点位高亮。

推荐操作流程：

1. 选择 PLC；
2. 选择分组；
3. 输入点位名称、显示名或地址关键字；
4. 查看卡片状态与质量。

## 6.3 PLC 调试 `PLC.Debug`

适用角色：`Engineer / Am`

页面分为两部分：

- 左侧：调试操作区
  - 按配置点位读写
  - 按地址直接读写
- 右侧：执行结果与历史表格

使用说明：

### 按配置点位读写

1. 选择 PLC；
2. 选择点位；
3. 查看地址、类型、长度；
4. 点击“读取”或输入写入值后点击“写入”。

### 按地址直接读写

1. 选择 PLC；
2. 输入地址、数据类型、长度；
3. 输入写入值；
4. 勾选“风险写入确认”；
5. 点击“读取”或“写入”。

注意：

- 高风险写入前应确认设备当前状态；
- 所有调试操作都会进入运行日志；
- 若写入失败，先检查地址、类型、长度和站连接状态。

## 6.4 PLC 配置 `SysConfig.Plc`

适用角色：`Engineer / Am`

主要功能：

- 站配置管理；
- 点位配置管理；
- 配置重载；
- 扫描控制；
- 搜索站 / 搜索点位。

推荐操作流程：

1. 先新增或编辑 PLC 站；
2. 选择一个站后再新增点位；
3. 保存后点击“重载配置”；
4. 进入 `PLC.Status` / `PLC.Monitor` 验证配置是否生效；
5. 需要联调时再进入 `PLC.Debug`。

## 8. 常用排查路径

### 8.1 PLC 点位无数据

排查建议：

1. `SysConfig.Plc` 检查站与点位配置；
2. 点击“重载配置”；
3. `PLC.Status` 检查站在线状态；
4. `PLC.Monitor` 检查点位质量；
5. `PLC.Debug` 做单点读取；
6. `RunLogPage` 查询错误日志。

### 8.2 轴动作失败

排查建议：

1. `Motion.Monitor` 查看轴报警和就绪状态；
2. `Motion.Axis` 检查当前轴与参数；
3. 查看当前报警页；
4. 查看运行日志。

### 8.3 用户看不到页面

排查建议：

1. 使用管理员进入 `System.Permission`；
2. 选择目标用户；
3. 检查该模块和页面是否勾选；
4. 保存后重新登录验证。
